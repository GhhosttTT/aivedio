**版本**：2.0  
**日期**：2026-04-28  
**作者**：Sarah (Product Owner)  
**质量得分**：96/100

## 执行摘要

AI 短剧自动化生产平台是一个端到端的短剧制作解决方案，旨在实现从创意到成品短剧的完全自动化。该平台通过集成 AI 剧本生成、文生图（Stable Diffusion）、图生视频（SVD）、AI 配音（MiMo-V2-TTS）和自动字幕生成技术，实现了只需输入主题或故事大纲即可自动生成完整短剧的能力。

该产品面向个人创作者，通过本地化部署和 Web 界面交互，大幅降低短剧制作的技术门槛和时间成本，使创作者能够专注于创意方向而非技术实现。预期可将短剧制作时间从数天缩短至数小时，并消除多工具切换的复杂性。用户只需输入主题或故事大纲，系统即可自动生成剧本、分镜、图像、视频、配音和字幕，最终输出可直接发布的短剧成品。

---

## 问题陈述

**当前情况**：短剧制作流程高度分散且门槛高，创作者需要：
- 手动编写剧本和分镜脚本（需要编剧能力）
- 在多个工具和平台之间切换
- 使用独立的文生图工具（如 Stable Diffusion WebUI）逐个生成场景图片
- 将图片导入图生视频工具（如 SVD）生成视频片段
- 使用配音工具为角色配音
- 手动添加字幕并调整时间轴
- 使用视频编辑软件合成完整短剧

这种分散性和高门槛导致：
- 大量时间浪费在工具切换和格式转换上
- 容易出现人为错误和遗漏
- 技术门槛高，需要掌握多个专业工具
- 创作门槛高，需要具备编剧能力
- 难以批量化和规模化生产

**提议解决方案**：构建一个统一的 AI 短剧自动化生产平台，通过 Web 界面接收主题或故事大纲输入，自动调用大语言模型生成剧本和分镜，再调度 ComfyUI（Stable Diffusion）、SVD 和 MiMo-V2-TTS 完成图像生成、视频生成、配音和字幕，最终输出完整的短剧成品。

**业务影响**：
- 制作效率提升 90% 以上（从数天缩短至数小时）
- 降低创作和技术双重门槛，使任何人都能制作高质量短剧
- 实现短剧制作的标准化和可复制性
- 为未来的批量化内容生产奠定基础
- 从"有想法"到"有成品"的时间缩短至小时级

## 成功指标

**主要 KPI：**

- **制作效率**：单集短剧（10-20 个镜头）从创意到成品的时间 < 3 小时（目标：2 小时）
  - 测量方法：记录从用户提交主题/大纲到系统输出最终视频的总耗时

- **自动化率**：无需人工干预即可完成的流程步骤占比 > 95%
  - 测量方法：统计需要人工介入的步骤数 / 总步骤数

- **成品质量**：生成的短剧视频在剧本质量、视觉连贯性、配音同步性、情感表达准确性方面的主观评分 > 4/5
  - 测量方法：创作者自评 + 小范围观众测试

**验证**：在 MVP 阶段完成 3-5 个完整短剧的制作，收集制作时间、自动化率和质量评分数据，验证系统是否达到预期目标。

## 用户画像

### 主要用户：个人短剧创作者

- **角色**：内容创作者、独立制片人、短视频博主
- **目标**：
  - 快速将创意想法转化为可发布的短剧视频
  - 降低制作成本和技术门槛
  - 提高内容产出效率，实现规模化创作
  - 无需编剧能力也能创作高质量短剧
- **痛点**：
  - 缺乏编剧能力，难以将想法转化为完整剧本
  - 缺乏专业的视频制作技能和设备
  - 多工具切换导致流程复杂、效率低下
  - 配音和视频合成需要大量手工操作
  - 难以保持多集短剧的视觉和风格一致性
- **技术水平**：初级到中级（熟悉基本的 Web 操作，但不具备深度技术背景）

## 功能需求

### 核心功能

**功能 0：AI 剧本与分镜自动生成**

- **描述**：通过大语言模型（LLM）自动生成完整剧本和分镜描述
- **用户流程**：
  1. 用户在 Web 界面选择"AI 生成剧本"
  2. 输入主题关键词或故事大纲（2-3 句话）
  3. 配置参数：集数、每集镜头数、角色数量、风格偏好
  4. 系统调用 LLM API（如 GPT-4、Claude、通义千问）生成剧本
  5. 系统自动解析剧本并生成分镜描述
  6. 用户可预览、编辑或重新生成
  7. 确认后进入生产流程
- **边界情况**：
  - 主题过于模糊时，提示用户补充更多细节
  - LLM 生成的剧本不符合预期时，支持重新生成或手动编辑
  - 生成的剧本过长或过短时，自动调整或提示用户
- **错误处理**：
  - LLM API 调用失败时，提示用户并提供重试选项
  - 生成超时（>2 分钟）时，提示用户并提供取消选项

**功能 1：剧本与分镜管理**

- **描述**：提供 Web 界面用于输入、编辑和管理剧本与分镜脚本（支持 AI 生成和手动输入两种模式）
- **用户流程**：
  1. 用户登录 Web 界面
  2. 创建新项目，选择"AI 生成"或"手动输入"
  3. AI 生成模式：输入主题/大纲，系统自动生成剧本和分镜
  4. 手动输入模式：在剧本编辑器中输入纯文本剧本，在分镜编辑器中输入分镜描述
  5. 保存草稿或提交生产任务
- **边界情况**：
  - 剧本或分镜为空时，提示用户补充内容
  - 分镜数量过多（>50）时，警告用户生成时间可能较长
  - 网络中断时，自动保存草稿到本地存储
- **错误处理**：
  - 输入格式错误时，高亮显示问题区域并提供修改建议
  - 保存失败时，提示用户并提供重试选项

**功能 2：ComfyUI 工作流集成（文生图）**

- **描述**：自动调用 ComfyUI 工作流，根据分镜描述生成高质量图像
- **用户流程**：
  1. 用户提交生产任务后，系统解析分镜描述
  2. 系统为每个分镜生成 Stable Diffusion 提示词（Prompt）
  3. 调用 ComfyUI API，传递提示词和生成参数
  4. ComfyUI 返回生成的图像，系统保存到本地存储
  5. 更新时间流状态为"图像生成完成"
- **边界情况**：
  - 分镜描述过于简短或模糊时，使用默认提示词模板补充
  - ComfyUI 生成失败时，自动重试最多 3 次
  - 生成的图像质量不符合预期时，允许用户手动调整提示词并重新生成
- **错误处理**：
  - ComfyUI 服务不可用时，提示用户检查 ComfyUI 是否正常运行
  - 生成超时（>5 分钟）时，提示用户并提供取消选项

**功能 3：SVD 集成（图生视频）**

- **描述**：自动调用 SVD 模型，将生成的图像转换为视频片段
- **用户流程**：
  1. 图像生成完成后，系统自动将图像传递给 SVD 模型
  2. SVD 根据配置的视频规格（分辨率、帧率、时长）生成视频
  3. 系统保存生成的视频片段到本地存储
  4. 更新时间流状态为"视频生成完成"
- **边界情况**：
  - 图像分辨率与 SVD 输入要求不匹配时，自动进行缩放或裁剪
  - SVD 生成失败时，自动重试最多 3 次
  - 视频时长不足时，使用循环或慢动作填充
- **错误处理**：
  - SVD 服务不可用时，提示用户检查 SVD 是否正常运行
  - 生成超时（>10 分钟）时，提示用户并提供取消选项

**功能 4：MiMo-V2-TTS 配音集成**

- **描述**：自动调用小米 MiMo-V2-TTS API，为剧本对话生成配音
- **用户流程**：
  1. 系统解析剧本中的对话内容和角色信息
  2. 用户为每个角色分配声音类型（男声、女声、方言、角色声音）和情感风格
  3. 系统调用 MiMo-V2-TTS API，生成配音音频
  4. 系统将配音音频与对应的视频片段进行时间对齐
  5. 更新时间流状态为"配音完成"
- **边界情况**：
  - 对话过长时，自动分段生成配音
  - 角色未分配声音时，使用默认声音
  - 配音时长与视频时长不匹配时，自动调整语速或视频播放速度
- **错误处理**：
  - MiMo-V2-TTS API 调用失败时，提示用户并提供重试选项
  - API 配额不足时，提示用户并暂停生产任务

**功能 5：白色字幕自动生成与嵌入**

- **描述**：根据剧本对话内容自动生成白色字幕并嵌入视频
- **用户流程**：
  1. 系统解析剧本中的对话内容
  2. 自动生成 SRT 字幕文件，时间轴与配音精确对齐
  3. 使用 FFmpeg 将字幕烧录到视频中（白色字体 + 黑色描边）
  4. 更新时间流状态为"字幕生成完成"
- **边界情况**：
  - 对话过长导致字幕显示不全时，自动分行或缩短显示时间
  - 字幕与配音时间轴不匹配时，自动调整字幕时间戳
- **错误处理**：
  - 字幕生成失败时，提示用户并提供重试选项
  - FFmpeg 烧录失败时，检查视频文件完整性并提示用户

**功能 6：视频合成与导出**

- **描述**：自动将所有视频片段、配音和字幕合成为完整的短剧成品
- **用户流程**：
  1. 所有镜头的视频、配音和字幕生成完成后，系统自动启动合成流程
  2. 系统按分镜顺序拼接视频片段
  3. 系统将配音音频与视频精确同步
  4. 字幕已嵌入视频中
  5. 用户可选择添加背景音乐
  6. 系统生成最终视频文件并提供下载链接
- **边界情况**：
  - 视频片段分辨率不一致时，自动统一为最高分辨率
  - 配音音频与视频时长不匹配时，自动调整音频播放速度或视频播放速度
  - 背景音乐时长不足时，自动循环播放
- **错误处理**：
  - 合成失败时，提示用户并提供重试选项
  - 导出失败时，检查磁盘空间并提示用户

**功能 7：时间流进度展示与预览**

- **描述**：在 Web 界面上展示每一集的生成进度时间流，支持悬浮预览
- **用户流程**：
  1. 用户提交生产任务后，时间流视图自动显示所有镜头的初始状态
  2. 随着生成进度推进，时间流实时更新每个镜头的状态（待处理、进行中、已完成）
  3. 用户鼠标悬浮在已完成的镜头上时，显示缩略图或短视频预览
  4. 用户点击镜头可查看完整的图像和视频结果
  5. 用户可对单个镜头进行重新生成操作
- **边界情况**：
  - 镜头数量过多（>50）时，时间流支持滚动和分页
  - 网络延迟导致状态更新不及时时，显示"正在同步..."提示
- **错误处理**：
  - 预览加载失败时，显示占位图并提供重新加载选项

### 不在范围内

- **多用户协作**：MVP 阶段仅支持单用户使用，不支持多人同时编辑同一项目
- **云端部署**：MVP 阶段仅支持本地部署，不提供云端 SaaS 服务
- **高级视频编辑**：不支持复杂的视频特效、转场动画、色彩调整等专业视频编辑功能
- **实时渲染**：不支持实时预览生成效果，需要等待生成完成后查看
- **移动端支持**：MVP 阶段仅支持桌面端 Web 界面，不支持移动端 App
- **多语言剧本生成**：MVP 阶段仅支持中文剧本生成

## 技术约束

### 性能

- **剧本生成时间**：单个剧本（10-20 个镜头）生成时间 < 1 分钟（基于 GPT-4 或同等性能 LLM）
- **图像生成时间**：单张图像生成时间 < 30 秒（基于 RTX 3090 或同等性能 GPU）
- **视频生成时间**：单个视频片段（3-5 秒）生成时间 < 2 分钟（基于 RTX 3090 或同等性能 GPU）
- **配音生成时间**：单句对话配音生成时间 < 5 秒（基于 MiMo-V2-TTS API）
- **字幕生成时间**：单集字幕生成时间 < 10 秒
- **视频合成时间**：10-20 个镜头的完整短剧合成时间 < 5 分钟

### 安全

- **数据隐私**：所有剧本、分镜、生成的图像和视频均存储在本地，不上传到云端（除 LLM API 和 MiMo-V2-TTS API 调用外）
- **API 密钥管理**：LLM API 密钥和 MiMo-V2-TTS API 密钥通过环境变量或配置文件管理，不硬编码在代码中
- **访问控制**：Web 界面支持基本的用户认证（用户名 + 密码），防止未授权访问

### 集成

- **Qwen2.5-14B**：本地部署的大语言模型，用于剧本生成
  - 依赖：需要下载 Qwen2.5-14B GGUF 格式模型（约 8-14GB，取决于量化级别）
  - 部署方式：使用 llama.cpp 进行推理，支持 GPU + CPU 混合推理（offload）
  - 硬件要求：GPU（RTX 4090/3090）+ CPU（多核心推荐）+ 至少 32GB RAM
  - 推理框架：llama.cpp + llama-cpp-python
  - 模型下载：https://huggingface.co/Qwen/Qwen2.5-14B-Instruct-GGUF
  - 量化选择：Q4_K_M（推荐，8GB）或 Q5_K_M（更高质量，10GB）

- **ComfyUI**：通过 ComfyUI 的 HTTP API 进行集成，支持自定义工作流
  - 依赖：ComfyUI 需要预先安装并运行在本地（默认端口 8188）
  - 工作流配置：支持通过 JSON 文件定义 Stable Diffusion 工作流

- **SVD**：通过 Python 脚本或 HTTP API 调用 SVD 模型
  - 依赖：SVD 模型需要预先下载并配置在本地
  - 支持的模型版本：Stable Video Diffusion 1.0 或更高版本

- **MiMo-V2-TTS**：通过小米官方 API 进行集成
  - 依赖：需要申请 MiMo-V2-TTS API 密钥
  - API 文档：https://mimo.xiaomi.com/mimo-v2-tts

### 技术栈

- **后端**：Python 3.10+
  - 框架：FastAPI（用于 Web API 和任务调度）
  - 任务队列：Celery + Redis（用于异步任务处理）
  - 视频处理：FFmpeg（用于视频合成、字幕嵌入和格式转换）
  - LLM 推理：llama.cpp（支持 GPU + CPU 混合推理，offload 到 CPU）
  - LLM 集成：llama-cpp-python（Python 绑定）

- **前端**：现代 Web 技术栈
  - 框架：React 18+ 或 Vue 3+
  - 状态管理：Redux 或 Pinia
  - UI 组件库：Ant Design 或 Element Plus
  - 实时通信：WebSocket（用于进度更新）

- **AI 模型**：
  - 剧本生成：Qwen2.5-14B（本地部署，后续可升级到 72B）
  - 文生图：Stable Diffusion（通过 ComfyUI，本地部署）
  - 图生视频：Stable Video Diffusion（SVD，本地部署）
  - 配音：小米 MiMo-V2-TTS（API 调用）

- **基础设施**：
  - 操作系统：Windows 10/11 或 Linux（Ubuntu 20.04+）
  - GPU：NVIDIA RTX 4090 24GB 或 RTX 3090 24GB（用于 Stable Diffusion + SVD）
  - CPU：多核心处理器（推荐 16 核心以上，用于 LLM offload）
  - 显存：至少 24GB（用于 Stable Diffusion + SVD，LLM 部分 offload 到 CPU）
  - 内存：至少 32GB RAM（推荐 64GB，用于 LLM CPU offload）
  - 存储：至少 500GB 可用空间（用于存储模型、图像和视频）
  - 模型存储：约 50GB（Qwen2.5-14B GGUF 8-14GB + SD 模型 10GB + SVD 模型 20GB）

### 兼容性

- **浏览器**：支持 Chrome 90+、Edge 90+、Firefox 88+（不支持 IE）
- **操作系统**：Windows 10/11、Linux（Ubuntu 20.04+）
- **GPU**：NVIDIA GPU（支持 CUDA 11.8+）

## 风险评估

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|----------|
| Qwen2.5-14B 生成的剧本质量不稳定 | 中 | 高 | 优化 Prompt 工程；提供剧本质量评分和重新生成选项；支持手动编辑；后续可升级到 72B 模型 |
| CPU offload 导致推理速度较慢 | 中 | 中 | 优化 offload 层数配置；使用高性能 CPU；增加 RAM 容量；考虑使用 GPU 加速 |
| ComfyUI 或 SVD 生成失败率高 | 中 | 高 | 实现自动重试机制（最多 3 次），并提供手动调整参数的选项；建立生成质量监控，记录失败原因 |
| MiMo-V2-TTS API 配额不足或服务不稳定 | 中 | 高 | 提前申请足够的 API 配额；实现 API 调用失败的降级方案（如使用备用 TTS 服务）；缓存已生成的配音音频 |
| GPU 性能不足导致生成时间过长 | 低 | 中 | 在文档中明确 GPU 性能要求（RTX 4090 或 RTX 3090）；提供性能优化建议（如降低分辨率、减少镜头数量） |
| 视频合成时音视频或字幕不同步 | 中 | 中 | 使用 FFmpeg 的精确时间戳对齐功能；在合成前进行音视频时长校验；提供手动调整同步偏移的选项 |
| 用户学习成本高，不会使用系统 | 低 | 中 | 提供详细的用户手册和视频教程；设计直观的 UI 界面；提供示例主题和剧本模板 |
