## 剧本创作

### 1. 提供详细的主题和大纲

**推荐做法**:
```json
{
    "theme": "友情、成长、温馨、励志、校园生活",
    "outline": "故事发生在一所小学，讲述两个性格迥异的好朋友小明（活泼开朗）和小红（内向细心）在面对学校运动会的挑战时，如何互相帮助、共同成长的故事。故事分为三个部分：1) 相遇与成为朋友；2) 面对运动会的挑战和分歧；3) 克服困难并取得成功。"
}
```

**不推荐做法**:
```json
{
    "theme": "友情",
    "outline": "两个朋友的故事"
}
```

**原因**: 详细的输入能帮助 AI 更好地理解您的创意，生成更符合预期的剧本。

### 2. 使用具体的关键词

**推荐的关键词类型**:
- **情感类**: 温馨、励志、悬疑、紧张、幽默、感人
- **风格类**: 现实主义、浪漫主义、魔幻现实、科幻
- **场景类**: 校园、都市、乡村、古代、未来
- **节奏类**: 快节奏、慢节奏、跌宕起伏

**示例**:
```
主题: 温馨、励志、校园生活、青春成长、友情
```

### 3. 控制剧本长度

**推荐分镜数量**:
- **短视频（1-3 分钟）**: 3-5 个分镜
- **中等视频（3-5 分钟）**: 5-8 个分镜
- **长视频（5-10 分钟）**: 8-12 个分镜

**原因**: 
- 每个分镜生成需要 5-15 分钟
- 太多分镜会导致生产时间过长
- 太少分镜可能无法完整表达故事

### 4. 明确角色设定

在大纲中明确角色的基本信息：

```
角色设定：
- 小明：10岁男孩，活泼开朗，擅长跑步，但不擅长跳远
- 小红：10岁女孩，内向细心，擅长跳远，但不擅长跑步
```

### 5. 设计清晰的故事结构

**推荐结构**:
1. **开端**（1-2 个分镜）：介绍角色和背景
2. **发展**（2-4 个分镜）：展开故事，引入冲突
3. **高潮**（1-2 个分镜）：冲突达到顶点
4. **结局**（1-2 个分镜）：解决冲突，总结主题

## 图像生成优化

### 1. 优化视觉提示词

系统会自动生成视觉提示词，但您可以通过调整剧本来影响生成结果。

**推荐的描述方式**:
```
场景描述：阳光明媚的校园操场，绿色的草坪上有红色的跑道，远处可以看到教学楼。小明穿着蓝色运动服，正在跑道上奔跑，脸上洋溢着自信的笑容。
```

**不推荐的描述方式**:
```
场景描述：小明在跑步。
```

### 2. 调整图像生成参数

在 ComfyUI 工作流配置中调整参数：

**高质量设置**（推荐）:
```json
{
    "steps": 30,
    "cfg_scale": 7.5,
    "width": 768,
    "height": 768,
    "sampler": "DPM++ 2M Karras"
}
```

**快速生成设置**（测试用）:
```json
{
    "steps": 20,
    "cfg_scale": 7.0,
    "width": 512,
    "height": 512,
    "sampler": "Euler a"
}
```

**参数说明**:
- `steps`: 采样步数，越高质量越好但速度越慢（推荐 20-50）
- `cfg_scale`: CFG 引导强度，控制图像与提示词的匹配度（推荐 7.0-9.0）
- `width/height`: 图像分辨率，越高质量越好但显存占用越大
- `sampler`: 采样器，不同采样器有不同的风格和速度

### 3. 保持风格一致性

**使用相同的种子值**:
- 在 ComfyUI 工作流中设置固定的种子值
- 可以确保同一项目的所有图像风格一致

**使用 LoRA 模型**:
- 可以添加特定风格的 LoRA 模型
- 例如：动漫风格、写实风格、水彩风格等

### 4. 处理生成失败

如果图像生成失败或质量不佳：

1. **检查提示词**：确保提示词清晰、具体
2. **调整 CFG Scale**：降低到 6.0-7.0
3. **增加采样步数**：提高到 30-40
4. **更换采样器**：尝试不同的采样器
5. **重新生成**：使用不同的种子值

## 视频生成优化

### 1. 调整视频参数

在 `.env` 文件中配置：

**高质量设置**（推荐）:
```bash
SVD_NUM_FRAMES=25
SVD_FPS=8
SVD_MOTION_BUCKET_ID=127
SVD_NOISE_AUG_STRENGTH=0.02
```

**快速生成设置**（测试用）:
```bash
SVD_NUM_FRAMES=16
SVD_FPS=8
SVD_MOTION_BUCKET_ID=127
SVD_NOISE_AUG_STRENGTH=0.02
```

**参数说明**:
- `SVD_NUM_FRAMES`: 视频帧数，越多越流畅但生成时间越长（推荐 16-25）
- `SVD_FPS`: 帧率，通常设置为 8
- `SVD_MOTION_BUCKET_ID`: 运动强度，越高运动越明显（推荐 100-150）
- `SVD_NOISE_AUG_STRENGTH`: 噪声增强强度，影响视频的多样性（推荐 0.02-0.05）

### 2. 优化图像输入

SVD 的视频质量很大程度上取决于输入图像：

- **使用高质量图像**：确保图像清晰、细节丰富
- **避免模糊图像**：模糊的图像会导致视频质量下降
- **保持主体居中**：主体应该在图像中心，便于生成动画

### 3. 控制运动幅度

**小幅度运动**（推荐）:
- 适合对话场景、静态场景
- `SVD_MOTION_BUCKET_ID=100-120`

**中等运动**:
- 适合行走、简单动作
- `SVD_MOTION_BUCKET_ID=120-140`

**大幅度运动**:
- 适合跑步、激烈动作
- `SVD_MOTION_BUCKET_ID=140-160`

### 4. 处理显存不足

如果遇到 GPU 显存不足：

1. **减少帧数**：从 25 降到 16
2. **降低分辨率**：在图像生成阶段降低分辨率
3. **启用 FP16**：在代码中启用半精度浮点数
4. **清理 GPU 缓存**：重启 Celery Worker

## 配音生成优化

### 1. 选择合适的说话人

MiMo-V2-TTS 提供多个说话人，选择合适的说话人可以提升配音质量：

**角色类型与说话人匹配**:
- **儿童角色**：选择音调较高、活泼的说话人
- **成年男性**：选择音调较低、沉稳的说话人
- **成年女性**：选择音调适中、温柔的说话人
- **老年角色**：选择音调较低、缓慢的说话人

### 2. 优化对话文本

**推荐做法**:
```
小明：你好，我叫小明！（停顿）很高兴认识你。
小红：你好，我叫小红。（停顿）我也很高兴认识你。
```

**不推荐做法**:
```
小明：你好我叫小明很高兴认识你
小红：你好我叫小红我也很高兴认识你
```

**优化技巧**:
- 使用标点符号控制停顿
- 避免过长的句子（建议每句不超过 30 字）
- 使用感叹号、问号表达情感

### 3. 调整语速和情感

在 TTS API 调用中可以调整参数：

```python
{
    "text": "你好，我叫小明！",
    "speaker": "speaker_1",
    "speed": 1.0,  # 语速（0.5-2.0，1.0 为正常）
    "emotion": "happy"  # 情感（happy, sad, angry, neutral）
}
```

### 4. 处理配额不足

如果 TTS API 配额不足：

1. **申请更多配额**：访问 MiMo 官网申请
2. **优化对话长度**：减少不必要的对话
3. **使用本地 TTS**：配置本地 TTS 服务（需要额外设置）

## 系统性能优化

### 1. GPU 显存优化

**单张 RTX 4090 24GB 配置**:

```bash
# LLM 配置（使用 CPU offload）
LLM_N_GPU_LAYERS=20  # 加载 20 层到 GPU，其余在 CPU
LLM_N_CTX=4096
LLM_N_THREADS=8

# GPU 显存管理
GPU_MEMORY_FRACTION=0.95
ENABLE_GPU_CACHE_CLEAR=true
```

**显存分配建议**:
- LLM: 4-6GB（使用 CPU offload）
- SD: 8-10GB
- SVD: 8-10GB
- 系统预留: 2-4GB

**优化策略**:
1. **串行执行 SD 和 SVD**：避免同时运行
2. **定期清理 GPU 缓存**：在任务之间清理
3. **使用 FP16 精度**：减少显存占用约 50%

### 2. CPU 和内存优化

**推荐配置**:
```bash
# Celery Worker 并发数
CELERY_WORKER_CONCURRENCY=2

# LLM CPU 线程数
LLM_N_THREADS=8
```

**优化建议**:
- 根据 CPU 核心数调整线程数
- 避免过高的并发数（会导致内存不足）
- 监控内存使用情况

### 3. 磁盘 I/O 优化

**使用 SSD**:
- 模型文件存储在 SSD 上
- 生成文件存储在 SSD 上
- 可以提升 20-30% 的性能

**定期清理**:
```bash
# 清理旧项目文件
python scripts/cleanup_old_projects.py --days 30
```

### 4. 网络优化

**Redis 优化**:
```bash
# Redis 配置
REDIS_URL=redis://localhost:6379/0
REDIS_MAX_CONNECTIONS=50
```

**TTS API 优化**:
- 使用连接池
- 启用重试机制
- 设置合理的超时时间

## 生产流程优化

### 1. 分阶段生产

**推荐流程**:
1. **第一阶段**：生成剧本，检查质量
2. **第二阶段**：生成 1-2 个分镜测试效果
3. **第三阶段**：调整参数后生成所有分镜
4. **第四阶段**：合成最终视频

**优点**:
- 及早发现问题
- 避免浪费资源
- 可以根据测试结果调整参数

### 2. 批量生产

如果需要生产多个项目：

**推荐做法**:
```python
# 串行生产（推荐）
for project_id in project_ids:
    submit_production_task(project_id)
    wait_for_completion(project_id)
```

**不推荐做法**:
```python
# 并行生产（会导致显存不足）
for project_id in project_ids:
    submit_production_task(project_id)
```

### 3. 监控和日志

**启用详细日志**:
```bash
LOG_LEVEL=INFO
LOG_FILE=./logs/app.log
```

**监控关键指标**:
- GPU 显存使用率
- CPU 使用率
- 内存使用率
- 磁盘空间
- 任务队列长度

**使用 Prometheus + Grafana**:
- 实时监控系统性能
- 设置告警规则
- 分析性能瓶颈

### 4. 错误处理和重试

**自动重试策略**:
```python
# Celery 任务配置
@app.task(
    bind=True,
    max_retries=3,
    default_retry_delay=60,
    autoretry_for=(Exception,)
)
```

**手动重试**:
- 查看任务失败原因
- 调整参数后重试
- 如果多次失败，检查系统配置

### 5. 备份和恢复

**定期备份**:
```bash
# 备份数据库
python scripts/backup_database.py

# 备份生成文件
python scripts/backup_storage.py
```

**恢复策略**:
- 保留最近 7 天的备份
- 重要项目单独备份
- 定期测试恢复流程

## 常见问题解决方案

### 问题 1：生成速度慢

**解决方案**:
1. 降低图像分辨率（768 → 512）
2. 减少视频帧数（25 → 16）
3. 减少采样步数（30 → 20）
4. 使用更快的采样器（DPM++ 2M → Euler a）
5. 升级硬件（更好的 GPU）

### 问题 2：生成质量不佳

**解决方案**:
1. 提供更详细的剧本描述
2. 增加采样步数（20 → 30-40）
3. 调整 CFG Scale（7.0 → 7.5-8.0）
4. 使用更好的采样器
5. 重新生成（使用不同的种子值）

### 问题 3：显存不足

**解决方案**:
1. 降低 LLM GPU 层数（20 → 15）
2. 降低图像分辨率
3. 减少视频帧数
4. 启用 FP16 精度
5. 清理 GPU 缓存
6. 串行执行任务

### 问题 4：任务频繁失败

**解决方案**:
1. 检查系统日志
2. 验证模型文件完整性
3. 检查网络连接（TTS API）
4. 增加重试次数
5. 调整超时时间

### 问题 5：视频不流畅

**解决方案**:
1. 增加视频帧数（16 → 25）
2. 使用更高质量的输入图像
3. 调整运动强度参数
4. 重新生成视频
