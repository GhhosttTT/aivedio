# LLM 服务性能基准数据

本文档记录 LLM 服务（Qwen2.5-14B）在不同配置下的性能基准数据，用于指导生产环境的配置选择。

## 测试环境

### 硬件配置

- **GPU**: NVIDIA RTX 4090 / RTX 3090 (24GB)
- **CPU**: Intel Core i7/i9 或 AMD Ryzen 7/9
- **内存**: 32GB+ DDR4/DDR5
- **存储**: NVMe SSD

### 软件配置

- **操作系统**: Windows 11 / Ubuntu 22.04
- **Python**: 3.10+
- **llama-cpp-python**: 0.2.20
- **CUDA**: 12.1+
- **模型**: Qwen2.5-14B-Instruct-Q4_K_M.gguf (~8GB)

## 性能测试方法

### 测试指标

1. **模型加载时间**: 从开始加载到模型就绪的时间
2. **显存占用**: 模型加载后增加的 GPU 显存使用量
3. **生成速度**: 每秒生成的字符数或 token 数
4. **生成质量**: 输出文本的连贯性和准确性（主观评估）

### 测试场景

1. **短文本生成**: 100 tokens 以内，测试响应速度
2. **中等文本生成**: 300 tokens 左右，测试稳定性
3. **长文本生成**: 500+ tokens，测试持续性能
4. **剧本生成**: 1000-2000 tokens，测试实际应用场景

### 测试配置

测试不同的 `n_gpu_layers` 配置：

- **0 层**: 纯 CPU 模式
- **10 层**: 轻量 GPU 模式
- **20 层**: 推荐配置（平衡性能和显存）
- **30 层**: 高性能模式
- **40 层**: 纯 GPU 模式（最高性能）

## 配置建议

### 生产环境推荐配置

基于测试结果，针对不同场景的推荐配置：

#### 场景 1: 多模型并行（LLM + SD + SVD）

**推荐配置**:
```env
LLM_N_GPU_LAYERS=20
LLM_N_CTX=4096
LLM_N_THREADS=8
```

**理由**:
- LLM 占用约 4-6GB 显存
- 为 SD 预留 8-10GB 显存
- 为 SVD 预留 8-10GB 显存
- 总计约 20-26GB，适合 24GB 显卡

#### 场景 2: 纯 LLM 应用

**推荐配置**:
```env
LLM_N_GPU_LAYERS=40
LLM_N_CTX=4096
LLM_N_THREADS=8
```

**理由**:
- 最高性能
- 不需要为其他模型预留显存

#### 场景 3: 显存紧张（16GB 显卡）

**推荐配置**:
```env
LLM_N_GPU_LAYERS=10
LLM_N_CTX=4096
LLM_N_THREADS=8
```

**理由**:
- 降低显存占用
- 为其他模型预留更多显存

#### 场景 4: 无 GPU 环境

**推荐配置**:
```env
LLM_N_GPU_LAYERS=0
LLM_N_CTX=2048
LLM_N_THREADS=16
```

**理由**:
- 纯 CPU 推理
- 增加 CPU 线程数以提高性能
- 减小上下文窗口以降低内存占用

## 性能优化建议

### 1. 模型选择

- **生产环境**: 使用 Q4_K_M 量化版本（平衡质量和性能）
- **高质量需求**: 使用 Q5_K_M 或 Q6_K 量化版本
- **低资源环境**: 使用 Q3_K_M 或更小的模型（Qwen2.5-7B）

### 2. 参数调优

- **temperature**: 0.7（推荐），平衡创造性和一致性
- **top_p**: 0.9（推荐），控制输出多样性
- **max_tokens**: 根据实际需求设置，避免过大
- **n_batch**: 512-1024，根据显存情况调整

### 3. 显存管理

- 在任务切换时调用 `torch.cuda.empty_cache()` 清理显存
- 使用单例模式避免重复加载模型
- 串行执行 SD 和 SVD 任务，避免显存冲突

### 4. 并发控制

- 限制同时运行的 LLM 请求数量
- 使用队列管理请求，避免显存溢出
- 考虑使用多进程或多实例部署

## 常见问题

### Q: 如何选择合适的 GPU 层数？

A: 根据以下因素决定：

1. **显存大小**: 24GB 显卡推荐 20 层，16GB 推荐 10-15 层
2. **并行模型**: 需要同时运行 SD/SVD 时，使用 15-20 层
3. **性能需求**: 对性能要求高时，使用 30-40 层

### Q: 为什么加载时间很长？

A: 可能的原因：

1. 首次加载需要初始化 CUDA 环境
2. 模型文件较大（~8GB）
3. 磁盘 I/O 速度慢（建议使用 NVMe SSD）

### Q: 如何提高生成速度？

A: 优化建议：

1. 增加 GPU 层数（如果显存充足）
2. 增加 `n_batch` 参数（如 1024）
3. 使用更小的模型（Qwen2.5-7B）
4. 减少 `max_tokens`

### Q: 显存不足怎么办？

A: 解决方案：

1. 减少 GPU 层数
2. 清理 GPU 缓存
3. 关闭其他占用显存的程序
4. 使用更小的模型
