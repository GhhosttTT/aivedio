# 当前 AI 生成短剧质量改造策略

## 结论

如果目标是接近红果短剧观感，不建议只靠当前本地 `SDXL + SVD` 链路继续调提示词。红果短剧的核心质感来自真人表演、真实场景、专业摄影、剪辑节奏和声音表演；当前项目是文生图和图生视频，天然会有表演僵硬、镜头运动假、角色漂移、情绪弱的问题。

推荐路线是混合方案：

```text
剧本/分镜/字幕/翻译：本地模型
关键图像/视频生成：高质量 API 或更强商业模型
批量低成本生成：本地 ComfyUI 工作流
最终合成/字幕/审核：本地
```

## 当前主要短板

1. 生成目标错位  
   当前系统生成的是“图片转短视频”，不是“真人短剧”。它很难自然呈现台词表演、走位、眼神、互动和连续动作。

2. 模型能力不足  
   SVD 更适合短运动片段，不适合复杂剧情表演。仅靠 `16 frames / 8 fps` 很难有红果短剧那种连贯镜头。

3. 提示词互相冲突  
   代码里同时存在“raw photo/candid shot/不完美”和“masterpiece/8k/flawless/professional photography”两套方向，会导致风格不稳定。

4. 角色一致性方案偏弱  
   只用首张生成图做参考图，容易把错误形象固化。应先建立角色定妆照，再用多角度参考控制后续镜头。

5. 没有剪辑语言  
   当前分镜更像静态画面描述，缺少短剧常见的钩子、反转、近景对话、情绪递进、节奏点和结尾悬念。

## API、本地、混合三种路线

| 路线 | 质量 | 成本 | 可控性 | 建议 |
|---|---|---|---|---|
| 纯本地 | 中低到中 | 低 | 高 | 适合内部批量测试，不适合直接追红果质感 |
| 纯 API | 高 | 高 | 中 | 适合验证商业质量，但成本不可控 |
| 混合 | 中高 | 中 | 高 | 推荐，关键镜头走 API，普通镜头走本地 |

## 推荐改造

### 第一阶段：让当前链路稳定

- 修复数据库字段、字幕任务、音频合成等确定性 bug。
- 固定一个真实感工作流，不在多个 workflow 间无策略切换。
- 把提示词策略统一成“短剧剧照/实拍感”，减少 `masterpiece`、`8k`、`flawless` 这类 AI 图味强的词。
- 增加角色定妆流程：每个角色先生成/上传 3 张参考图，再生产剧情镜头。

### 第二阶段：提升短剧感

- 剧本生成改为“短剧商业结构”：前三秒冲突、每 15-30 秒反转、每集结尾钩子。
- 分镜增加镜头类型：特写、过肩、双人对话、反应镜头、插入镜头。
- 视频生成不要依赖 SVD 做长表演，改为短镜头拼接。
- TTS 改成可控情绪和多角色音色，优先 API，后续再本地化。

### 第三阶段：接高质量视频 API

保留本地平台，但增加 provider 抽象：

```text
local_comfyui
api_image
api_video
hybrid
```

关键镜头、主角特写、高潮镜头走 API；背景镜头、转场镜头、低价值镜头走本地。

## 短期决策

先不要把当前系统强行改成纯本地模型追红果质量。更稳的选择是：

1. 源片出海功能走本地模型，因为它处理已有实拍源片，质量上限来自源片。
2. AI 生成短剧功能走混合模式，因为从零生成真人短剧对本地模型要求过高。
3. 当前本地生成链路保留为低成本草稿和分镜预览，不作为最终发行级成片。
