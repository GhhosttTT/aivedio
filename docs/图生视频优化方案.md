# 🎬 图生视频优化方案

## 📊 当前方案分析

### 现有方案: Stable Video Diffusion (SVD)

**优点**:
- ✅ 开源免费
- ✅ 质量不错
- ✅ 已集成到项目中

**缺点**:
- ❌ 显存占用大 (8-10GB)
- ❌ 生成速度慢 (30-60秒/视频)
- ❌ 只能生成 16 帧 (2秒视频)
- ❌ 运动幅度有限
- ❌ RTX 3060 可能吃力

### 当前配置

```python
num_frames = 16      # 16 帧
fps = 8              # 8 帧/秒 = 2秒视频
motion_bucket_id = 127  # 运动强度 (1-255)
noise_aug_strength = 0.02  # 噪声强度
```

## 🌟 市面上更好的图生视频方案

### 方案 1: AnimateDiff ⭐⭐⭐⭐⭐ (强烈推荐!)

**特点**:
- ✅ 显存占用低 (6-8GB)
- ✅ 生成速度快 (15-30秒)
- ✅ 可生成更长视频 (24-48 帧)
- ✅ 运动更自然流畅
- ✅ 与 SDXL 完美集成
- ✅ **RTX 3060 友好!**

**工作原理**:
- 在 SDXL 基础上添加运动模块
- 可以直接在 ComfyUI 中使用
- 支持 ControlNet 控制运动

**下载**:
- 模型: https://huggingface.co/guoyww/animatediff
- ComfyUI 节点: https://github.com/Kosinkadink/ComfyUI-AnimateDiff-Evolved

**配置示例**:
```json
{
  "model": "animatediff_xl_beta.ckpt",
  "frames": 24,
  "fps": 12,
  "motion_scale": 1.0,
  "context_length": 16
}
```

## 🎯 推荐方案: AnimateDiff

### 为什么选择 AnimateDiff?

1. **RTX 3060 友好**
   - 显存占用: 6-8GB (比 SVD 少 2-3GB)
   - 生成速度: 15-30秒 (比 SVD 快 50%)

2. **质量更好**
   - 运动更流畅自然
   - 可生成更长视频 (24-48 帧)
   - 与 SDXL 完美集成

3. **灵活性强**
   - 支持 ControlNet 控制运动
   - 可调整运动幅度
   - 支持循环视频

### AnimateDiff 工作流

#### ComfyUI 工作流结构

```
CheckpointLoader (SDXL)
  ↓
AnimateDiffLoader (运动模块)
  ↓
AnimateDiffModelSettings (运动参数)
  ↓
LoadImage (输入图像)
  ↓
CLIPTextEncode (提示词)
  ↓
KSampler (生成视频帧)
  ↓
VAEDecode
  ↓
VHS_VideoCombine (合成视频)
  ↓
SaveVideo
```

## 💡 提示词优化

### 运动提示词

好的运动提示词可以显著提升视频质量:

```python
# ✅ 好的运动提示词
"smooth camera pan, cinematic motion, slow zoom in, professional cinematography"

# ✅ 特定运动
"camera slowly moves forward, gentle zoom, steady shot"

# ✅ 人物动作
"person walking naturally, smooth movement, realistic motion"

# ❌ 避免
"fast motion, quick cuts, shaky camera"  # 可能导致不稳定
```
