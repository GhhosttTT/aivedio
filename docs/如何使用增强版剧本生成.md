# 如何使用增强版剧本生成

## 问题背景

你提出的观察非常准确：**文生图的不清晰很可能是因为剧本生成的不够详尽导致的**。

当前系统的问题：
1. 剧本描述过于简单（每个分镜只有 50-100 字）
2. 缺少章节结构，剧情连贯性不足
3. 没有完整的故事大纲
4. 角色描述不够详细，导致角色一致性差

## 解决方案

我已经创建了**增强版剧本生成方案**，核心思想是：**把剧本当作一本书来写，分章节组织，每个分镜都有详细的描述**。

### 关键改进

1. **增加故事大纲**（200-300字）
   - 背景设定
   - 主要冲突
   - 情节发展
   - 结局走向

2. **增加章节结构**（3-5章）
   - 每章有独立的情节推进
   - 每章有章节概述（100-150字）
   - 每章结尾有情节转折

3. **增加分镜详细描述**（每个分镜200+字）
   - 故事节点（30-50字）：说明本分镜在故事中的作用
   - 环境描述（至少50字）：时间、地点、天气、光线、具体物品、背景元素、空间布局
   - 人物描述（至少40字）：位置、姿势、表情、动作、服装细节、肢体语言、情绪状态
   - 镜头描述（至少20字）：景别、拍摄角度、构图、焦点、景深
   - 光线描述（至少20字）：光源类型、位置、色温、明暗对比、阴影效果
   - 氛围描述（至少20字）：情绪基调、视觉风格、色彩倾向、质感、电影感

4. **增加角色详细设定**（每个角色100+字）
   - 年龄性别
   - 外貌特征（详细描述五官、发型、身材、肤色）
   - 服装风格（详细描述服装款式、颜色、材质、配饰）
   - 性格特点
   - 角色定位

我已经创建了 `src/services/enhanced_script_prompt.py` 文件，你可以直接使用：

    num_scenes=12,
    num_characters=2,
    style="现代都市",
    num_chapters=3  # 新增参数：章节数量
)

# 使用 LLM 生成剧本
from src.services.llm_service import get_llm_service

llm_service = get_llm_service()

```

在 `src/services/llm_service.py` 中，将 `generate_script_prompt` 方法替换为增强版：

# 在 llm_service.py 顶部导入
from src.services.enhanced_script_prompt import generate_enhanced_script_prompt

# 在 LLMService 类中修改方法
def generate_script_prompt(

    num_characters: int = 2,

    构建剧本生成的 Prompt（使用增强版）

```

### 方法 3：在 API 中使用

修改 `src/services/script_generator.py` 中的 `generate_script` 方法，增加 `num_chapters` 参数：

```python
def generate_script(
    self,
    project_id: int,
    theme: Optional[str] = None,
    outline: Optional[str] = None,
    num_scenes: int = 10,
    num_characters: int = 2,
    style: str = "现代都市",
    num_chapters: int = 3,  # 新增参数
    temperature: float = 0.7,
    max_tokens: int = 8192  # 增加默认值
) -> Dict:
    """生成剧本"""
    # 构建 Prompt（使用增强版）
    from src.services.enhanced_script_prompt import generate_enhanced_script_prompt
    
    prompt = generate_enhanced_script_prompt(
        theme=theme,
        outline=outline,
        num_scenes=num_scenes,
        num_characters=num_characters,
        style=style,
        num_chapters=num_chapters
    )
    
    # 调用 LLM 生成剧本
    script_text = self.llm_service.generate(
        prompt=prompt,
        max_tokens=max_tokens,
        temperature=temperature
    )
    
    # 解析剧本（需要增强解析逻辑以支持新格式）
    parsed_script = self.parse_script(script_text)
    
    return parsed_script
```

## 注意事项

### 1. LLM Token 消耗增加

增强版 Prompt 更长，生成的内容也更多，会消耗更多 token：

- **Prompt 长度**：从 1000 字增加到 3000+ 字
- **生成内容**：从 2000 字增加到 5000+ 字
- **建议**：将 `max_tokens` 从 4096 增加到 8192

### 2. 生成时间增加

更详细的内容需要更长的生成时间：

- **之前**：约 30-60 秒
- **现在**：约 60-120 秒
- **建议**：在 UI 中显示更详细的进度提示

### 3. 解析逻辑需要增强

新格式包含更多字段，需要增强解析逻辑：

- 解析【故事大纲】
- 解析【章节】结构
- 解析新增的字段（故事节点、章节位置等）

### 4. 数据库模型需要扩展

建议增加以下表和字段：

```python
# 新增章节表
class Chapter(Base):
    __tablename__ = "chapters"
    
    id = Column(Integer, primary_key=True)
    project_id = Column(Integer, ForeignKey("projects.id"))
    chapter_number = Column(Integer)
    title = Column(String(200))
    summary = Column(Text)  # 章节概述
    
# 扩展分镜表
class Scene(Base):
    # 新增字段
    chapter_id = Column(Integer, ForeignKey("chapters.id"))
    chapter_position = Column(String(50))  # 例如："第一章 开场"
    story_node = Column(Text)  # 故事节点描述
    environment_detail = Column(Text)  # 详细环境描述
    character_detail = Column(Text)  # 详细人物描述
    camera_detail = Column(Text)  # 详细镜头描述
    lighting_detail = Column(Text)  # 详细光线描述
    atmosphere_detail = Column(Text)  # 详细氛围描述
```

1. ✅ 生成完整的故事大纲（200-300字）
2. ✅ 按章节组织剧本（3-5章）
