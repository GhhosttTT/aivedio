# 视频生产流程修复说明

## 📋 问题概述

在之前的实现中,视频生产流程存在以下问题:

1. **任务链参数传递错误**: 使用了 `.s()` 方法但任务函数包含 `previous_result` 参数,导致参数不匹配
2. **任务无法正确执行**: 图像生成后,后续的视频、音频、字幕生成任务无法正确接收参数
3. **前端缺少独立控制**: 只能一键启动全流程,无法单独触发各个步骤

## ✅ 已完成的修复

### 1. 修复任务编排器 (`src/services/task_orchestrator.py`)

**问题**: 使用 `.s()` 方法创建任务签名,会将前一个任务的返回值作为第一个参数传递

**解决方案**: 改用 `.si()` 方法创建不可变签名,忽略前一个任务的返回值

```python
# ❌ 修改前
generate_video_task.s(scene.id, project_id, task_id)

# ✅ 修改后
generate_video_task.si(scene.id, project_id, task_id)
```

**影响的任务**:
- 图像生成任务 ✅
- 视频生成任务 ✅
- 音频生成任务 ✅
- 字幕生成任务 ✅
- 合成任务 ✅

## ❓ 常见问题

### Q1: 为什么要使用 `.si()` 而不是 `.s()`?

**A**: 
- `.s()` 创建普通签名,会将前一个任务的返回值作为第一个参数传递
- `.si()` 创建不可变签名,忽略前一个任务的返回值,只使用指定的参数
- 在我们的场景中,每个任务都有明确的参数,不需要前一个任务的返回值

### Q2: 任务链中的并行任务如何工作?

**A**: 
- 使用 `group()` 创建并行任务组
- 同一个 `group` 中的所有任务会并行执行
- 只有当整个 `group` 完成后,才会执行下一个步骤

### Q3: 如果某个分镜的任务失败了怎么办?

**A**: 
- 当前实现中,如果某个任务失败,整个任务链会停止
- 可以使用"重新制作"功能重新启动流程
- 未来可以添加单个任务重试功能

### Q4: 如何查看任务执行的详细日志?

**A**: 
```bash
# Celery Worker 日志
tail -f logs/celery.log

# 或者在启动 worker 时查看实时输出
celery -A src.tasks.celery_app worker --loglevel=debug
```

### Q5: 前端如何知道任务进度?

**A**: 
- 前端通过 WebSocket 连接到后端
- 每个任务完成时会通过 WebSocket 推送进度更新
- 前端实时更新 UI 显示
